# Product Requirements Document (PRD) - Sad Phonebook

## 1. 프로젝트 개요
웹 기반의 전화번호 관리 애플리케이션으로, Supabase 데이터베이스와 Vercel 호스팅을 통한 간단하고 직관적인 전화번호 관리 시스템입니다.

## 2. 주요 기능

### 2.1 전화번호 입력 (Create)
- 사용자가 새로운 전화번호 정보를 입력할 수 있음
- 입력 항목: 이름, 전화번호 (최소 필수 항목)
- 입력 폼을 통한 데이터 추가

### 2.2 전화번호 조회 (Read)
- 저장된 모든 전화번호 목록 조회
- 테이블 또는 리스트 형식으로 표시
- 각 항목의 상세 정보 확인

### 2.3 전화번호 수정 (Update)
- 기존 전화번호 정보 수정 가능
- 이름, 전화번호 등의 정보 변경
- 수정 폼을 통한 데이터 업데이트

### 2.4 전화번호 삭제 (Delete)
- 불필요한 전화번호 정보 삭제
- 삭제 확인 절차 포함

## 3. 비기능 요구사항

### 3.1 로그인 및 인증
- 로그인 기능 불필요
- 누구나 접근 가능한 공개 애플리케이션

### 3.2 기술 스택
- **프론트엔드**: 웹 기반 (HTML, CSS, JavaScript 또는 현대적 프레임워크)
- **백엔드/데이터베이스**: Supabase (PostgreSQL 기반)
- **호스팅**: Vercel

### 3.3 UI/UX
- 기본적인 기능에 집중한 심플한 디자인
- 직관적인 네비게이션
- 모바일 친화적 반응형 디자인 권장

## 4. 데이터 모델

### 4.1 전화번호 테이블 (contacts)
```
- id (Primary Key)
- name (문자열, 필수)
- phone_number (문자열, 필수)
- created_at (타임스탐프)
- updated_at (타임스탐프)
```

## 5. 사용자 경험 흐름

1. 애플리케이션 접속
2. 저장된 전화번호 목록 조회
3. 새 번호 추가 / 기존 번호 수정 / 번호 삭제
4. 변경사항이 Supabase에 자동으로 저장

## 6. 구현 범위

### 포함 사항
- 기본 CRUD 기능 (Create, Read, Update, Delete)
- Supabase 데이터베이스 연동
- Vercel 배포 설정

### 제외 사항
- 사용자 인증/로그인
- 고급 검색/필터링 기능
- 통화 기록 등 추가 기능
- 복잡한 UI/UX 디자인

## 7. 성공 기준

- ✓ 전화번호 추가 기능 정상 작동
- ✓ 전화번호 수정 기능 정상 작동
- ✓ 전화번호 삭제 기능 정상 작동
- ✓ Supabase와 정상 연동
- ✓ Vercel에서 정상 배포 및 접속 가능
